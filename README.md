# PolicyPal

> An AI-powered chatbot that answers insurance questions from a curated knowledge base, using Retrieval-Augmented Generation (RAG) over PostgreSQL + pgvector.

PolicyPal lets a logged-in user ask natural-language questions about any kind of insurance — health, auto, life, home, and more — and returns grounded answers generated by an LLM using context retrieved from a vector store. Source content is ingested from Wikipedia, cleaned, chunked, embedded, and stored in pgvector for semantic search.

## Features

- **Conversational Q&A** over a curated insurance knowledge base
- **RAG pipeline** — semantic retrieval from pgvector feeds an LLM for grounded, context-aware answers
- **User authentication** — JWT-based login; only authenticated users can query
- **Reproducible ingestion** — a single command runs download → clean → chunk → embed → store
- **Typed end to end** — Pydantic schemas on the backend, TypeScript on the frontend

## Architecture

```
Wikipedia ──► Ingestion pipeline ──► pgvector
                                        │
User ──► React UI ──► FastAPI ──► Retrieval ──► LLM ──► Answer
                         ▲                                │
                         └──────── grounded response ─────┘
```

The system has two halves that share a common core (embeddings + chunking):

- **Offline ingestion** populates the vector store (run periodically).
- **Online serving** retrieves relevant chunks per query and generates an answer.

Both must use the same embedding model — enforced by sharing `core/embeddings.py`.

## Getting Started

### Prerequisites

- Python 3.12+ and [uv](https://github.com/astral-sh/uv)
- Node.js 20.19+ (for the frontend)
- Docker (for PostgreSQL + pgvector)

### Setup

```bash
# 1. Install dependencies
make install

# 2. Start PostgreSQL + pgvector
make db-up

# 3. Apply database migrations (creates schema + enables pgvector)
make migrate

# 4. Run the ingestion pipeline to populate the vector store
make ingest

# 5. Start the backend (http://localhost:8000)
make api

# 6. In a separate terminal, start the frontend (http://localhost:5173)
make frontend
```

Copy `.env.example` to `.env` and fill in your database URL and LLM API key before running.

## Environment Variables

| Variable        | Description                          |
| --------------- | ------------------------------------ |
| `DATABASE_URL`  | PostgreSQL connection string         |
| `LLM_API_KEY`   | API key for the language model       |
| `SECRET_KEY`    | Secret for signing JWT tokens        |

## Ingestion Pipeline

The pipeline runs in ordered stages and is resumable:

```bash
# full run
uv run python -m policypal.ingestion.pipeline

# resume from a later stage after a failure
uv run python -m policypal.ingestion.pipeline --from-stage embed
```

**download** → fetch Wikipedia articles · **clean** → normalize text ·
**chunk** → recursive splitting · **embed** → vectorize and upsert to pgvector
